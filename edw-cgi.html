At CGI Inc., I was responsible for building and expanding an Enterprise Data Warehouse for government contracting clients operating in regulated environments. The work required Kimball dimensional modeling discipline — conformed dimensions, fact tables at consistent grains, and SCD Type 2 for tracking historical changes accurately. The EDW needed to support both operational reporting and analytical workloads across multiple client programs.

My direct contributions included source data profiling and onboarding, SCD2 dimension design and implementation, two-point data quality gate architecture, ETL streamlining (achieving 35% processing time reduction), and SSIS framework standardization. I collaborated in Agile ceremonies and mentored junior developers on SSIS and SQL best practices.

Design Context

The Kimball star schema provided a consistent, report-friendly layer across all client programs. Conformed dimensions (Program, Org, Date, Geography) ensured the same 'Program' meant the same thing across all reports — eliminating the inconsistency that plagues ad-hoc warehouse builds.

02 Phase 1 — Source Onboarding & Data Profiling
✦ I Directly Led This

Integrating a new data source into a production EDW requires profiling it first — uniqueness of natural keys, null rates on required fields, allowed value domains, and timestamp reliability for incremental loading. Skipping this step causes silent data quality issues that poison downstream reports.

T-SQL — Source Data Profiling PatternsSimulated pattern — no proprietary code
-- Null-rate profile for required fields
SELECT
  SUM(CASE WHEN ProgramCode IS NULL THEN 1 ELSE 0 END) AS Null_ProgramCode,
  SUM(CASE WHEN OrgCode IS NULL THEN 1 ELSE 0 END)     AS Null_OrgCode,
  COUNT(*)                                              AS TotalRows
FROM src.ProgramActivity;

-- Natural key uniqueness check
SELECT ProgramCode, COUNT(*) AS c
FROM src.Program
GROUP BY ProgramCode
HAVING COUNT(*) > 1;
03 Phase 2 — Dimensional Modeling & SCD Type 2
✦ I Directly Built This

The EDW used Kimball dimensional modeling — facts at a consistent grain, conformed dimensions with surrogate keys, and SCD Type 2 for any dimension where historical accuracy matters. I designed and implemented the SCD2 MERGE pattern that preserves history without overwriting it.

T-SQL — SCD Type 2 MERGE PatternSimulated pattern
-- Dimension table with SCD2 columns
CREATE TABLE dw.DimProgram (
  ProgramSK     int IDENTITY(1,1) PRIMARY KEY,
  ProgramCode   nvarchar(50) NOT NULL,
  ProgramName   nvarchar(200) NULL,
  Status        nvarchar(30) NULL,
  EffectiveFrom date NOT NULL,
  EffectiveTo   date NOT NULL,
  IsCurrent     bit NOT NULL,
  HashDiff      binary(32) NULL
);

-- SCD2 merge: expire changed rows, insert new current versions
DECLARE @AsOf date = CAST(GETDATE() AS date);

-- Expire changed current rows
UPDATE d
SET d.EffectiveTo = DATEADD(day,-1,@AsOf), d.IsCurrent = 0
FROM dw.DimProgram d
JOIN stg.Program s ON s.ProgramCode = d.ProgramCode
WHERE d.IsCurrent = 1
  AND d.HashDiff != HASHBYTES('SHA2_256',CONCAT(s.ProgramName,'|',s.Status));

-- Insert new current rows
INSERT INTO dw.DimProgram(ProgramCode,ProgramName,Status,EffectiveFrom,EffectiveTo,IsCurrent,HashDiff)
SELECT s.ProgramCode, s.ProgramName, s.Status,
       @AsOf, '9999-12-31', 1,
       HASHBYTES('SHA2_256',CONCAT(s.ProgramName,'|',s.Status))
FROM stg.Program s
LEFT JOIN dw.DimProgram d ON d.ProgramCode=s.ProgramCode AND d.IsCurrent=1
WHERE d.ProgramCode IS NULL
   OR d.HashDiff != HASHBYTES('SHA2_256',CONCAT(s.ProgramName,'|',s.Status));
04 Phase 3 — Data Quality Controls (+25% Accuracy)
✦ I Directly Built This

I embedded validation at two points: staging (reject invalid rows before they reach the warehouse) and fact loading (referential integrity check before inserting any fact row whose dimension keys don't resolve). This two-point quality gate is what drives the 25% accuracy improvement.

T-SQL — Two-Point Quality Gate PatternSimulated pattern
-- Gate 1: Domain validation in staging
INSERT INTO etl.Rejects(EntityName,NaturalKey,RejectReason,LoadBatchID)
SELECT 'Program', ProgramCode,
  CONCAT('Invalid Status: ', COALESCE(Status,'NULL')), @BatchID
FROM stg.Program
WHERE COALESCE(Status,'') NOT IN ('Active','Inactive','Pending');

-- Gate 2: Referential integrity before fact insert
SELECT DISTINCT s.ProgramCode
FROM stg.ProgramActivity s
LEFT JOIN dw.DimProgram p
  ON p.ProgramCode=s.ProgramCode AND p.IsCurrent=1
WHERE p.ProgramSK IS NULL;  -- these will be rejected
05 Phase 4 — ETL Streamlining (−35% Processing Time)
✦ I Directly Led This

I profiled each pipeline step to identify where time was being spent — slow source extracts, blocking SSIS transforms, or heavy destination loads. Indexing source extraction predicates and enabling parallelism for independent entity loads were the two highest-impact changes.

T-SQL — Extraction Index & Batch Load PatternSimulated pattern
-- Index to speed source extraction (watermark-based incremental)
CREATE NONCLUSTERED INDEX IX_src_UpdatedAt
ON src.ProgramActivity (UpdatedAt)
INCLUDE (ProgramCode, OrgCode, ActivityCount);

-- Batched fact insert for large datasets
DECLARE @BatchSize int = 200000;
WHILE 1=1
BEGIN
  INSERT INTO dw.FactActivity(DateKey,ProgramSK,OrgSK,ActivityCount,LoadBatchID)
  SELECT TOP(@BatchSize)
    d.DateKey, p.ProgramSK, o.OrgSK, s.ActivityCount, @BatchID
  FROM stg.ProgramActivity s
  JOIN dw.DimDate d    ON d.[Date]=s.BusinessDate
  JOIN dw.DimProgram p ON p.ProgramCode=s.ProgramCode AND p.IsCurrent=1
  JOIN dw.DimOrg o     ON o.OrgCode=s.OrgCode AND o.IsCurrent=1
  WHERE s.IsLoaded=0;
  IF @@ROWCOUNT=0 BREAK;
END
